## Operation-cost model for first iteration

Input-size variables:

- `T`: number of traces.
- `N`: total events.
- `A`: number of distinct activity labels.
- `D`: number of observed direct-follows pairs.
- `V`: number of distinct variants.
- `L`: maximum trace length.
- `P`: prefix states or trie nodes when an automaton candidate is used.
- `k`: fixed bound on preset/postset candidate size.
- `M`: fixed maximum loop-body length for length-bounded rework-loop candidates.

Allowed counted primitive operations:

- `scan_event`: reading one event or one adjacent event pair during log summarization.
- `dict_increment`: incrementing or recording one counter/map entry.
- `set_insert`: adding one element to a set-like structure.
- `set_lookup`: membership or keyed lookup in a set/map relation.
- `comparison`: equality, threshold, subset, or ordering decision.
- `arithmetic`: one elementary numeric operation used for scores or margins.
- `relation_classification`: assigning a relation/dependency class from counted evidence.
- `construct`: creating a PMIR item, place, transition, arc, marking, or emitted pattern.

Uncounted in this first iteration: Python interpreter overhead, sorting for stable JSON, JSON serialization, dataclass/list membership overhead inside `PetriNet`, and import/module startup. This is a research operation model, not a CPU profiler.

Candidate expected costs under this model:

- `ALG-0001`: `O(N + A^2 + A^(2k) + C^2)` for bounded Alpha pair enumeration and subsumption over candidate pairs `C`.
- `ALG-0002`: `O(N + A^2 + local_branch_checks)` in the implemented full-pair score variant; an observed-edge-only variant could be `O(N + D)`.
- `ALG-0003`: current non-recursive prototype is `O(N + A^2 + T * L^2)` for DFG/relation summaries plus bounded cut checks; a future recursive version is expected around `O(N + A^3)` if decomposition checks remain bounded.
- `ALG-0004`: `O(N + P + A^(2k) * P * k)` with small fixed `k` and no external solver.
- `ALG-0005`: trie build `O(N)`, suffix-signature compression `O(P * signature_cost)`, and automaton-net construction `O(P + E)` over compressed states and automaton edges.
- `ALG-0006`: `O(N + A^2 + sum_v out(v)^2 + sum_v in(v)^2)` for relation classification and local split/join grouping.
- `ALG-0009`: `O(N + A^2 + sum_v out(v)^2 + sum_v in(v)^2 + optional_checks)` with local optional checks bounded by outgoing-degree pairs.
- `ALG-0010`: `O(N * L + A^2 + D^2 + path_checks + local_branch_checks)` for eventual-order evidence, transitive reduction, optional-chain selection, and residual guards.
- `ALG-0011`: `O(N + P + A^(2k) * P * k + A + E)` with the `ALG-0004` bounded region cost plus causal-map construction and non-overlapping optional singleton checks.
- `ALG-0012`: `O(N + P + A^(2k) * P * k + N * L + A^2 + D^2 + path_checks)` for bounded region enumeration plus optional-chain detection, order consistency, and selected-shortcut certification.
- `ALG-0013`: same as `ALG-0012` minus selected-shortcut certification checks; measured savings are currently tiny because chain detection and bounded-region enumeration dominate.
- `ALG-0014`: abstraction path `O(N + A^2 + T * L^2 + B)` for common-boundary block checks and block construction, where `B` is emitted block size; fallback path adds the `ALG-0005` exact automaton cost `O(P * signature_cost + P + E)`.
- `ALG-0015`: `ALG-0014` plus support-count/skew checks, optional prefix-merge checks, same-activity-set dominant-sequence checks, and exact fallback when no grammar is selected.
- `ALG-0016`: same grammar checks as `ALG-0015` but exact fallback is disabled; no-grammar outputs construct only the base rejecting net.
- `ALG-0017`: `ALG-0014` plus support-count/skew checks only; exact fallback remains enabled.
- `ALG-0018`: `ALG-0017` plus prefix-merge segment construction/checking; dominant-sequence checks are disabled.
- `ALG-0019`: `ALG-0017` plus same-activity-set dominant-sequence checks; prefix-merge checks are disabled.
- `ALG-0020`: same checks as `ALG-0015`, but common-boundary detection is tried before prefix merge and prefix merge is skipped when common-boundary detection succeeds.
- `ALG-0021`: `ALG-0015` plus a second bounded common-boundary/prefix-merge check to emit ambiguity alternatives into PMIR evidence.
- `ALG-0022`: `ALG-0021` discovery plus bounded block-net construction for each emitted ambiguity alternative; measured totals include discovery plus per-alternative compile counts.
- `ALG-0023`: `ALG-0003` plus `O(T * L + A)` singleton rework-loop detection and constant-size duplicate-labeled loop-net construction.
- `ALG-0024`: `ALG-0023` plus a bounded multi-body loop detector with body-support counts, expected `O(T * L + B)` for `B` singleton body alternatives, and body-choice loop-net construction.
- `ALG-0025`: `ALG-0024` with body alternatives bounded by `M=2`, expected `O(T * L * M + B * M)` additional validation and sequence-body construction for fixed `M`; measured counts include rejected-detector overhead before singleton-loop fallback.
- `ALG-0026`: `ALG-0025` discovery plus one bounded at-most-once alternative compiler, expected `O(B * M + |prefix| + |suffix|)` additional construction operations for fixed `M=2`; selected-run operation counts remain the upstream unbounded net for benchmark compatibility.
- `ALG-0027`: `ALG-0026` policy-set discovery plus validation replay over two alternatives and `O(P + Vp + Vn)` selector comparisons/arithmetic for `P` alternatives, `Vp` validation positives, and `Vn` validation negatives; EXP-0029 reports selector primitive counts plus a validation replay proxy of one `scan_event` per validation event and one `comparison` per validation trace per alternative.
- `ALG-0028`: `ALG-0025` discovery plus `O(B)` body-support policy checks; if filtering applies, add `O(T * L)` trace/body partitioning and `O(B_kept * M + |prefix| + |suffix|)` guarded loop-net compilation for fixed `M=2`.
- `ALG-0029`: `ALG-0025` keep-all discovery plus `ALG-0028` support-guard discovery, validation replay over two alternatives, `O(P + Vp + Vn + T)` selector comparisons/arithmetic for `P=2`, and validation replay proxy counts. Selector outputs retain naive all-alternative discovery totals and, since EXP-0038, also include shared-accounting totals as one `ALG-0025` base plus support-guard incremental extra, selector counts, and validation proxy counts.
- `ALG-0030`: `ALG-0029` body-axis selection plus count-policy compilation from the selected body result, count-validation replay over at most two alternatives, and `O(P_count + Vp_count + Vn_count)` selector scoring. Selector outputs retain naive product totals and, since EXP-0038, also include conservative shared product totals using the shared body axis plus all count-policy compile extras, count selector cost, and count validation replay proxy counts.
- `ALG-0031`: same as `ALG-0028` with `rare_body_count=2` and a five-of-seven dominant-share policy: `ALG-0025` discovery plus `O(B)` support checks and, when filtering applies, `O(T * L)` body partitioning plus bounded recompilation.
- `ALG-0032`: `ALG-0025` discovery plus `O(B)` dominance/rare-body checks and `O(2^R * (T * L + compile_guarded_net + validation_proxy))` bounded per-body alternatives, where `R <= max_rare_bodies` and the prototype refuses selection when the cap is exceeded. Selector outputs retain naive all-alternative upper bounds and, since EXP-0038, also include shared totals by charging the `drop_none` base once plus each non-base incremental guarded-recompile extra.
- `ALG-0033`: `ALG-0025` discovery plus `O(B)` dominance/rare-body checks, one `O(V * L)` validation parsing pass, bounded unit-clause checks over rare bodies in validation negatives, at most one `O(T * L + compile_guarded_net)` guarded recompilation, and one selected-net validation replay proxy. It records avoided exhaustive alternatives but does not build or replay all `2^R` assignments.

First limited-operation budget tiers for subsequent promotion discussions:

- Smoke soft budget: `B_smoke = 10N + 8A^2 + 6D + 80` counted primitive operations on a toy log. A candidate can exceed this as a baseline, but low-operation advantage claims should say so explicitly.
- Deep-test soft budget: `B_deep = 16N + 12A^2 + 10D + 120` counted primitive operations on synthetic logs with at most five distinct activities.
- Promotion note: these budgets are screening thresholds, not quality metrics. Replay, negative probes, structural diagnostics, and documented scope still dominate promotion decisions.

- `ALG-0006` PMIR Split-Join Compiler: novel local PMIR split/join compiler. First smoke gate: sequence, XOR, parallel, skip; short-loop/noise as counterexample gates.
